Roadmap

v0.1: Control Plane

Status: current scaffold.

  • Initialize durable harness files.
  • Create raw/ as the user-owned context folder.
  • Enforce immutable goal and protected policy files through PreToolUse.
  • Track tasks through TASKS.json.
  • Track host command templates through ADAPTERS.json.
  • Track machine events through .yoloop/events.jsonl.
  • Provide the first host adapter as a Claude Code plugin wrapper.

v0.2: Local Runner

  • Add yoloop run.
  • Spawn one worker session per task.
  • Spawn one critic session after each worker handoff.
  • Support configurable command templates for Claude Code, Codex, and other agent hosts.
  • Capture stdout, stderr, exit code, duration, and transcript paths into events.jsonl.
  • Stop on human gates, budget exhaustion, or critic rejection threshold.

v0.3: Critic Verdict Enforcement

  • Add yoloop critic write-verdict.
  • Validate verdicts against schemas/critic-verdict.schema.json.
  • Only allow completed transitions when the latest verdict is approved.
  • Require deterministic check evidence for approval.

v0.4: Worktree Isolation

  • Run each task in an isolated git worktree.
  • Merge approved task branches back through the harness.
  • Detect overlapping path ownership.
  • Reject unsafe parallelism unless tasks have disjoint write scopes.

v0.5: Parallel Workers

  • Add bounded worker pool.
  • Assign tasks by dependency graph and path ownership.
  • Run per-task critics before integration.
  • Run grand jury after all tasks complete.

v1.0: Overnight Mode

  • End-to-end yoloop run --until-done.
  • Host adapters for Claude Code, Codex, OpenCode, and Cursor-like CLIs.
  • Restart-safe run state.
  • Final <yoloop-done> emission only after grand jury approval.
  • Human-readable morning report generated from events, progress, failures, decisions, and verdicts.
